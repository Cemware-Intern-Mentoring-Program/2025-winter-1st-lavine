> 주제 3: Spring은 무엇일까? 또, Spring과 SpringBoot의 주요한 차이점은 뭘까?

> - Spring은 어떤 구성요소를 포함하고 있는가?
> - Spring과 SpringBoot의 주요 차이점은?

## Spring은 어떤 구성요소를 포함하고 있는가?

Spring은 다양한 기능을 모듈 형태로 제공한다.
대표적인 핵심 구성요소와 역할을 간단히 정리하면 다음과 같다

- Core / Bean Container : IoC/DI 기반 객체 관리(스프링 컨테이너)
- AOP 모듈 : 횡단 관심사 분리
- Spring MVC : 웹 애플리케이션 개발 지원(`DispatcherServlet` 기반)
- Spring JDBC/ORM : 데이터 접근 추상화(`JPA`, `Hibernate` 연동)
- Spring Security : 인증/인가 처리
- Spring Test : 테스트 편의 기능 지원

이러한 다양한 모듈을 필요할 때 원하는 모듈만 선택하여 사용할 수 있는 것이 스프링의 강점이다.

## Spring과 SpringBoot의 주요 차이점은?

스프링은 다양한 모듈을 통해 많은 기능을 지원하지만, 그 설정이 복잡하다. 

스프링부트는 **이러한 스프링을 더 쉽게 사용할 수 있게 해주는 도구로, 설정 보다 개발에 집중할 수 있도록 자동화를 제공한다.**

즉, 스프링이 엔터프라이즈 개발을 위한 강력한 기반을 제공하고 스프링 부트는 그 기반을 효율적으로 활용하게 해주는 생산성 프레임워크를 제공한다.

| 항목      | **Spring (Framework)**  | **Spring Boot**              |
| ------- | ----------------------- | ---------------------------- |
| 설정 방식   | XML/Java 설정 직접 작성       | 자동 설정(Auto Configuration) 지원 |
| 서버 실행   | 별도 WAS 설치 필요 (Tomcat 등) | 내장 서버 제공 -> 실행만 하면 됨          |
| 프로젝트 설정 | 의존성 직접 추가 및 관리          | Starter 의존성으로 간편 구성          |
| 관점      | 프레임워크 (기반 제공)           | 개발 생산성 극대화 도구                |
| 목표      | 유연성과 확장성                | 빠르게 만들고 바로 실행                |
