## 요청(Request)와 응답(Response)은 뭘까?

HTTP 통신은 클라이언트가 요청을 보내고 서버가 응답을 보내는 구조다.
- 요청에는 메서드(무엇을 할 것인지), URI(어디에 보낼지), 헤더&바디(추가정보)를 담아서 보낸다.
- 응답에는 처리 결과를 담아 클라이언트에게 돌려준다.

즉 서버는 요청에 대한 **결과 + 설명 + 메타정보**를 함께 전달해야 한다.

응답에 담길 수 있는 값은 다음과 같다.
- StatusCode : 요청 처리 결과(성공,실패 등)
- Header : 캐싱 여부, 길이, 인증 정보 등 부가정보
- Body : 실제 데이터(JSON,XML,HTML 등)

## ResponseEntity
> HTTP 상태 코드 + 헤더 + 바디를 모두 개발자가 직접 구성할 수 있는 응답 wrapper

즉, 응답을 가장 정제된 형태로 제어할 수 있는 스프링의 응답 메커니즘이다. 

다음과 같은 형태로 사용한다. 

```Java
return ResponseEntity
        .status(HttpStatus.CREATED)
        .header("Location", "/members/1")
        .body(memberResponse);
```

### 사용하는 이유
ResponseEntity를 통해 상태 코드, 헤더, 바디를 설정을 직접 세밀하게 조정하고 반환할 수 있기 때문에, 클라이언트 측이 보낸 요청에 대해 RESTful한 API를 작성할 수 있다.
